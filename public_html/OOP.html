Overview of OOP (object-oriented programing) & Pylint


• OOP ( object oriented programing) : In Python, object-oriented Programming (OOPs) is a programming concept that uses objects and classes in programming. It aims to implement real-world entities like inheritance, polymorphisms, encapsulation, etc. in the programming. The main concept of OOPs is to bind the data and the functions that work on that together as a single unit so that no other part of the code can access this data.

    • Class & Object

  • A class is a collection of objects. A class contains the blueprints or the prototype from which the objects are being created.
  • The object is an entity that has a state and behavior associated with it. It may be any real-world object like a mouse, keyboard, chair, table, pen, etc. Integers, strings, floating-point numbers, even arrays, and dictionaries, are all objects.

    • Pylint

  • Pylint is a tool that checks for errors in Python code, tries to enforce a coding standard.It can also look for certain type errors, it can recommend suggestions about how particular issues can be solved and can offer you details about the code's complexity.

    • what is Pytest?

  • Pytest is a test framework in python. To install pytest, we need to use the command pip install pytest. After installation, we can verify if python has been installed by the command pytest –version

    • Glossary

  • A namespace- is a collection of currently defined symbolic names along with information about the object that each name references. You can think of a namespace as a dictionary in which the keys are the object names and the values are the objects themselves. Each key-value pair maps a name to its corresponding object.
  • Constructors in Python- Constructors are generally used for instantiating an object. The goal of constructors is to assign values to the data members of the class when an object of the class is created. In Python the __init__() method is called the constructor.
  • Type Hinting- In simple word, type hinting means providing hints to function to only accept the given data type.
  • Type Casting- We are allowed to transform our data type up through casting.
  • Unit Testing- is the first level of software testing, where the purpose is to validate that each unit in the program performs as designed.
  • Static Methods- is marked with a @staticmethod decorator to flag it as a static method. This type of method takes neither a self nor a cls parameter.
  • Facade- is a structural design pattern that provides a simplified but limited interface to a complex system of classes, library or framework.
  • Design patterns- there are many design patterns that are used in programming. Factory method is one of them.
  • Method- a method is a code that only runs when it is called in object-oriented programming, a method is defined within a class
  • Property – property refers to the special values included in a class or object
