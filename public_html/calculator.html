How to Implement design patterns - Separation of Concerns


• Separation of concerns is a software architecture design pattern/principle for separating an application into distinct sections, so each section addresses a separate concern. In basics, Separation of concerns is about order. The overall goal of separation of concerns is to establish a well-organized system where each part fulfills a meaningful role.

• Separation of concerns is achieved by the establishment of boundaries

• if all of the code for a particular behavior of the application is separated out, then you will only have to change code directly associated with your new feature. Which should be less code to change.


    Applying SoC to the Calculator Program

    Functions Class

  • For each of the calculator's operations, we created function classes. A class for addition, subtraction, multiplication, and division is involved. Having separate classes for these functions simplifies coding significantly. In the history class, for example, each function can simply be called from the class. The functions class, like the calculation class, hides a lot of complexity, making the code simpler and easier to manage.

    History Class

  • We can save the history of the calculations in a list using the history class. This class also allows for the addition of calculations, the clearing of history, and the return of the number of calculations in the history. Other useful functions include determining the value or object type of a specific calculation in the history.

    Calculator Class

  • The calculator is the final result or the final product that will be used to implement all of the functions written before it. The calculator class will automatically add calculations to a calculator history while also providing other functions such as showing calculation history size and clearing history. Because we used separation of concerns, the final calculator class will appear less complex with potential free of error.

    Calculation class

  • Finally, there is a calculation class we created for the calculator program and it is related to calculator class which includes all the value input needed for the calculation. We use this class every time we need to call a function by defining the parameters for the calculator's inputs in one class. This also saves time because we wouldn't have to write the parameter conditions for each function class if we did have this class. This is not only effective, but it also makes each function class appear more complicated than necessary.
