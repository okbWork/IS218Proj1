Article 1: Pylint and Glossary


  • Pylint allows you to do proper testing for your python programs as well as make sure that your programs are in line with modern PEP8 convention, meaning that you use clean and understandable syntax!

  • Its best to install and use this package using docker to make sure theres no errors across different systems and platforms.
  • In your Requirements.txt make sure to include the following to install the necessary things to lint, cover, and test your files
  • In your Dockerfile make sure to include this command to install the required files for pylint
  • Use this command to run your pylint to check your code: "pylint --pytest --cov"
  • the flag --cov can be used to see how much of your code the tests and pylint check
  • If there are things that you dont want to check, you can use this style of comment in order to make sure that its skipped in the pylint process.
  • Here is a Glossary For all important OOP Terms used in the calculator program:
    • Term/Design Patt. Definition Example
    Factory An object used to create other objects. The Calculator class in calc2.1/calc/calculator.py is a Factory because it creates new operation classes when its called (Addition, Subtraction, Division, and Multiplication Classes)
    Instantiation The realization/creation of an object instance. A simple statement like "x = 1.5" is an instantiation because it takes a float object and assigns it a name (x) and a value (1.5)
    Object An encapsulation of data and procedures to manipulate the data A creation of a calculation from calc2.1/calculations/calculation.py is an object of the calculation class
    Class A blueprint for creating objects calculation in calc2.1/calculations/calculation.py is a class. It is used to create Calculation objects
    Namespace A way to specify which library is being used in case there are conflicting library terms While this isnt common in python, namespaces in C++ are used with common libraries like std in order to easily and neatly access the namespaces methods and classes
    Constructor A method used to create an object The "__init__" method in calc2.1/calculations/calculation.py is a constructor because it is whats executed when the class is called to make a new object
    Fixture In testing: it is a way to prep the environment for testing in calc2.1/tests/calculations_history_test.py the method clear_history_fixture clears the history of the calculator to make sure that the results of a test wont be ruined by any previous calculations
    Type Hint In implicit typed languages like PHP and Python, type hinting is a way of explicitly casting a variable In calc2.1/calculations/calculation.py on line 5, the parameter "values: tuple" creates a variable "values" but also demands that the variable be a tuple object
    Type Cast Used to convert or declare a variable as a very specific type while type casting varies based on types and language, an example in python would be "x = int(y)" where y could be a decimal like 5.0 or a string like "5" but because of the int statement surrounding it, it is converted directly to an integer
    Unit Test A way of testing methods and classes The tests in calc2.1/tests/division_test.py use unit tests to try specific cases and ensure that even if there an error there will be no breaks in the program
    Static Anything that is static is accessible anywhere and has a lifetime of the entire program the "convert_args_to_list_float" method in calc2.1/calculations/calculation.py is a static method meaning that it can always be accesed at any point in the programs
    Class Method A method that exists solely within another class in calc2.1/calculations/addition.py, the "get_result" method is a class method that is only usable thorugh that class or an instance of that class
    Facade An encapsulation of multiple other classes, variables, and methods The calculator class acts as a facade for all of the other classes and its encapsulated methods and objects
    Method A series of operations and actions used to manipulate and create data the methods within calc2.1/tests/addition_test.py are just methods that arent class methods because they can be called without needing a class instantiation and perform a series of operations and actions
    Property A variable characteristic of a class In calc2.1/calculations/calculation.py, the values attribute is a property of the Calculation class because it is a variable in that class.
